# Brain Tumor Classification Using Deep Learning

A deep learning system that classifies brain tumors from MRI scans into four categories — **glioma, meningioma, pituitary tumor, and no tumor** — using a CNN built on the EfficientNetB0 backbone, with an interactive web app for real-time predictions.

Final year B.E. project — Department of Computer Science & Engineering (Data Science), Sathyabama Institute of Science and Technology.

## Overview

Manual interpretation of MRI scans is time-consuming and can be inconsistent across radiologists. This project automates brain tumor classification using transfer learning, so a user can upload an MRI image and get an instant, reliable prediction.

## Features

- **4-class classification**: Glioma, Meningioma, Pituitary tumor, No tumor
- **Transfer learning** with EfficientNetB0 as the feature extraction backbone
- **Data augmentation** (rotation, flipping, zooming) to improve generalization
- **Real-time predictions** through a Streamlit web interface
- Model evaluation using accuracy, confusion matrix, precision/recall

## How It Works

1. **Upload** — User uploads an MRI image via the Streamlit interface.
2. **Preprocess** — Image is resized, normalized, and converted into a tensor.
3. **Predict** — The trained EfficientNetB0-based CNN classifies the image.
4. **Display** — The predicted tumor class is shown alongside the uploaded image.

## Model Training

- Optimizer: Adam
- Loss: Categorical cross-entropy
- Regularization: Early stopping + model checkpointing to prevent overfitting
- Evaluation metrics: Accuracy, Confusion Matrix, Precision, Recall, F1-score

## Results

The model achieves strong, consistent classification performance across all four tumor classes, with good generalization on unseen validation data (see `docs/Project_Report.pdf` for full experimental results, confusion matrix, and precision-confidence curves).

## Training the Model

The dataset is expected in the standard 4-class MRI folder layout (e.g. the Kaggle
"Brain Tumor MRI Dataset"):

```
data/Training/{glioma,meningioma,notumor,pituitary}/
data/Testing/{glioma,meningioma,notumor,pituitary}/
```

Then train with:

```bash
python -m src.train
```

This saves the best model to `model/brain_tumor_effnet.keras` and a training
curve plot to `model/training_history.png`, and prints a classification
report + confusion matrix on the validation set.
